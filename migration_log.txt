Source: app/app.py @ 13,004 lines on origin/main (post PR #275)

Rules followed for every move:
  - No logic changes; lines are lifted as-is, with a `# moved from app.py:NNN`
    marker above each block.
  - `from <module> import *` style; app/__init__.py NOT introduced.
  - state.py has an explicit __all__ list so underscore-prefixed names
    survive `from state import *`.

Names INTENTIONALLY left in app.py (would have broken if moved):
  - _logger        — used 100+ times inside app.py; moving and re-importing
                     adds noise with no benefit (user decision).
  - _werkzeug_main — runtime guard, not state (user decision).
  - _sched         — initial declaration line 1358, BUT reassigned at boot
                     line 12290 (`_sched = nightly_retrain.start()`).  Under
                     `from state import *`, the reassignment would only
                     rebind app.py's local name and state.py's _sched would
                     still be None.  Kept in app.py.
  - _STAT_LOOKUP_LOG_BUDGET — reassigned via `global` inside
                     _reset_stat_lookup_log_budget() (line 10776) and
                     _stat_lookup_log() (line 10854).  Same rebind problem
                     as _sched.  Kept in app.py.
  - _delete_file   — pure helper, but depends on _logger.  Would have moved
                     to utils.py; rerouted to "stay in app.py" because the
                     alternative was a circular import or a rewrite.
  - _audit_log     — same situation: depends on _eprint, which depends on
                     _redact + sys.stderr.  Kept in app.py to avoid pulling
                     _eprint along.
